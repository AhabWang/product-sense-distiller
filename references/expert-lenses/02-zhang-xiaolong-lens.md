# Zhang Xiaolong Lens

This file defines the Zhang Xiaolong Lens for Product Sense Distiller.

This lens does not imitate Zhang Xiaolong's voice.

It extracts a product thinking perspective inspired by Zhang Xiaolong-style product judgment:

- Restraint
- Low interruption
- User psychology
- Natural interaction
- Respect for user autonomy
- Product as a quiet tool, not a loud presence
- Less, but more precise

Use this lens only as an auxiliary perspective.

The final judgment must return to Wang Product Philosophy.

---

## 1. Core Question

The Zhang Xiaolong Lens asks:

> Does this product respect the user, or is it trying to occupy, disturb, control, and over-educate the user?

This lens is especially skeptical of:

- Over-design
- Over-notification
- Over-interaction
- Over-explanation
- Over-personalization
- Over-gamification
- Unnecessary social pressure
- Products that make users do more work instead of less

Useful judgment:

> 好产品不应该总是跳出来证明自己存在。真正好的工具，常常是安静地帮用户完成事情。

---

## 2. When to Use This Lens

Use Zhang Xiaolong Lens when evaluating:

- App interaction
- Notification systems
- AI companions
- Chat interfaces
- Sleep products
- Emotional products
- Social products
- Content products
- User onboarding
- Reminder systems
- Products that risk becoming too heavy
- Products that may disturb the user

Especially use it when the question is:

- Is this feature over-designed?
- Is AI conversation too much?
- Is the app too heavy?
- Is the product interrupting the user?
- Is the product making users think too much?
- Should we cut functions?
- Does the product respect the user's original behavior?

---

## 3. Core Product Beliefs

### 3.1 Restraint is a product capability

Restraint is not weakness.

Restraint means knowing what not to do.

Ask:

- What should this product not do?
- What should it not ask from the user?
- What should it not interrupt?
- What should it not explain?
- What should it not gamify?
- What should it not measure?
- What should it not push?

Useful judgment:

> 克制不是少做，而是不把产品团队的欲望转嫁给用户。

---

### 3.2 A product should not demand unnecessary attention

User attention is expensive.

Do not treat attention as free.

Ask:

- Does this feature require active attention?
- Does it interrupt the user's current state?
- Does it ask users to check something frequently?
- Does it create unnecessary reminders?
- Does it make users anxious if they ignore it?
- Does it turn a simple task into a management task?

Useful judgment:

> 产品越想证明自己重要，越容易变成用户的负担。

---

### 3.3 Natural behavior is stronger than forced behavior

A product should fit into the user's natural behavior.

Do not force users to perform unnatural rituals unless the value is strong enough.

Ask:

- What is the user's original behavior?
- What does the product add?
- What does the product remove?
- Does the product make the behavior smoother?
- Or does it create a new burden?
- Does the product require users to learn a new mental model?
- Is the behavior change worth it?

Useful judgment:

> 好产品不是让用户配合产品，而是让产品贴近用户原本的行为。

---

### 3.4 Less interaction can mean better experience

More interaction is not always better.

Especially in sleep, baby care, health, and tool products.

Ask:

- Can this step be removed?
- Can this be automatic?
- Can this be hidden?
- Can this be inferred?
- Can this be done without opening the app?
- Can this be done without screen interaction?
- Can this be completed without a decision?

Useful judgment:

> 交互不是越多越好。很多时候，少一步，就是好体验。

---

### 3.5 User psychology matters more than feature logic

Feature logic can be internally correct but psychologically wrong.

Ask:

- How does the user feel when seeing this?
- Does this create pressure?
- Does this create trust?
- Does this create anxiety?
- Does this make the user feel judged?
- Does this make the user feel accompanied?
- Does this make the user feel controlled?
- Does this make the product feel warm or intrusive?

Useful judgment:

> 功能上成立，不代表心理上舒服。

---

## 4. Key Evaluation Questions

When using Zhang Xiaolong Lens, ask:

### Restraint

- What is the product overdoing?
- Which feature exists because the team wants it, not because users need it?
- What should be cut?
- What should be delayed?
- What should be hidden?
- What should never be done?

### Low Interruption

- Does the product interrupt users?
- Does it notify too often?
- Does it require unnecessary attention?
- Does it make users check status too frequently?
- Does it create anxiety when users do not respond?

### User Psychology

- Does the product make users feel respected?
- Does it make users feel watched?
- Does it create pressure?
- Does it reduce cognitive load?
- Does it align with the user's emotional state?

### Natural Interaction

- Does the product fit existing user behavior?
- Does it create new behavior burden?
- Does the interaction feel natural?
- Can the user understand it without instruction?
- Can the user stop using it without feeling trapped?

### Product Boundary

- Is the product trying to do too much?
- Is it crossing from useful to intrusive?
- Is it trying to own too much of the user's time?
- Is the product becoming heavier than the problem?

Useful judgment:

> 张小龙 Lens 不问“还能加什么”，而是问“这个产品有没有资格打扰用户”。

---

## 5. Red Flags

Zhang Xiaolong Lens should be skeptical when seeing:

- Too many notifications
- Too many reminders
- Too many onboarding steps
- Too many modes
- Too many settings
- Too much gamification
- Too much forced personalization
- App required for basic use
- AI conversation that does not know when to stop
- Health data that creates anxiety without action
- Sleep product that requires screen interaction
- Emotional companion that becomes needy
- Product that makes users manage the product
- Product that uses guilt, pressure, or FOMO to drive behavior
- Product that over-educates users before value is felt

Useful judgment:

> 如果一个产品需要用户长期管理它，它就不是在帮用户减负，而是在制造新负担。

---

## 6. Zhang Xiaolong Lens Output Structure

When the user asks to apply Zhang Xiaolong Lens, use this structure:

1. Zhang Xiaolong Lens verdict
2. Where the product is restrained
3. Where the product is over-designed
4. User psychology analysis
5. Interaction and interruption risk
6. What to cut
7. What should remain quiet
8. Return to Wang Product Philosophy for final judgment

Do not imitate Zhang Xiaolong's voice.

Do not make vague statements about minimalism.

Be specific about which feature, interaction, notification, or product behavior should be reduced.

---

## 7. Example Application: Bedtime Companion

When applying Zhang Xiaolong Lens to Bedtime Companion:

### Core question

Is Bedtime Companion a quiet bedtime companion, or is it becoming an overactive AI toy?

### What to protect

- Low-stimulation bedtime experience
- Hug-first interaction
- Breathing rhythm
- Gentle presence
- Minimal screen dependency
- Conversation that knows when to stop

### What to be careful about

- AI talks too much
- Too many conversation modes
- Too much emotional probing
- Too many reminders
- App-based sleep management
- Sleep reports that create anxiety
- Over-personalization that feels creepy
- Turning bedtime into another task

### Zhang Xiaolong Lens judgment

Bedtime Companion should not try to win by talking more.

Its value may come from knowing when not to talk.

Useful judgment:

> 对 Bedtime Companion 来说，最好的 AI 不是更会聊天，而是更懂什么时候安静。

---

## 8. Example Application: Bottle Washer

When applying Zhang Xiaolong Lens to a baby bottle washer:

### Core question

Does the product reduce caregiver burden, or does it create a new management burden?

### What to protect

- Simple loading
- One-button operation
- Clear status feedback
- Minimal app dependency
- Low maintenance
- No unnecessary notifications
- No complex modes

### What to be careful about

- Too many cleaning programs
- App control that is not necessary
- Push notifications that parents do not need
- Status details that create anxiety
- Maintenance steps that feel frequent
- Making tired parents learn a machine

### Zhang Xiaolong Lens judgment

The best bottle washer experience is not “smart.”

It is:

> Put bottles in, press once, get clean and dry bottles out.

Useful judgment:

> 母婴产品的克制，是不要把父母已经很累的生活再变复杂。

---

## 9. Example Application: Health / Wearable Products

When applying Zhang Xiaolong Lens to health wearables:

### Core question

Does the product help users understand themselves, or does it make users more anxious?

### What to protect

- Clear actionable insight
- Calm presentation
- Minimal false alarm
- User control over notifications
- Honest uncertainty
- Simple interpretation

### What to be careful about

- Too many scores
- Too many warnings
- Over-precise health claims
- Daily judgment that makes users feel bad
- Data without action
- Notifications that interrupt life
- Turning health into constant self-surveillance

### Zhang Xiaolong Lens judgment

Health data should reduce uncertainty, not create a new anxiety loop.

Useful judgment:

> 健康产品最怕把“理解身体”做成“每天被数据审判”。

---

## 10. How to Combine with Wang Product Philosophy

Zhang Xiaolong Lens helps reduce over-design and user disturbance.

But final judgment must return to Wang Product Philosophy.

After using this lens, still ask:

- Is the demand strong?
- Is the value anchor hard enough?
- Does restraint improve purchase and retention?
- Does cutting features weaken or strengthen the product?
- Does the product still have a must-buy reason?
- Can the hardware and business still work?

Useful judgment:

> 张小龙 Lens 可以帮我们做减法；王师傅主脑要继续判断减法之后，这个产品还够不够硬、值不值得买、能不能成立。

---

## 11. Final Zhang Xiaolong Lens Rule

When uncertain, use this rule:

If a feature increases user attention, cognitive load, emotional pressure, or behavior burden without clearly improving value, cut it, hide it, or delay it.

A good product should not always ask users to look at it.

It should help users live better with less effort.
